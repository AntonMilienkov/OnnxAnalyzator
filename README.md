# OnnxAnalyzator

Анализатор моделей формата ONNX состоит из трех компонент: анализ весов, анализ графа модели и сравнение моделей.

## Анализ весов
Анализ весов выполняется для всех параметров модели и позволяет вычислять следующие характеристики:

Базовые свойства:
1. имя тензора
2. форма
3. тип данных
4. размер — общее количество элементов

Статистические характеристики:

5. минимальное значение
6. максимальное значение
7. среднее значение
8. стандартное отклонение

Нормы:

9. L1-норма 
10. L2-норма

Проверка на аномалии:

11. количество значений NaN
12. количество бесконечных значений (Inf)
13. проверка на нулевой тензор
Эти метрики используются для выявления ошибок обучения или повреждений модели.


Суммирующие метрики по модели:
14. общее количество весов
15. суммарное количество параметров
16. списки весов с NaN, Inf и полностью нулевых

## Анализ графа
Анализ графа:

Общие свойства графа:
1. количество нод
2. количество входных тензоров
3. количество выходных тензоров
4. количество промежуточных тензоров
5. количество параметровых тензоров

Типы операций:
6. гистограмма операций (количество каждого типа op_type)

Структура графа:
7. соответствие producer (кто создаёт тензор)
8. соответствие consumer (кто использует тензор)
9. граф зависимостей между нодами (adjacency)
10. количество входящих связей у ноды (in-degree)

Топология графа:
11. топологический порядок выполнения нод
12. наличие циклов в графе 

Глубина графа:
13. глубина каждой ноды 
14. максимальная глубина графа 

Специальные типы узлов:
15. ветвящиеся узлы (имеют несколько потребителей)
16. листовые узлы (не используются далее)
17. узлы-источники (не имеют входящих зависимостей)

Характеристики отдельных нод:
18.1. тип операции
18.2. входные тензоры
18.3. выходные тензоры
18.4. количество входящих связей
18.5. количество исходящих связей
18.6. глубина


## Сравнение моделей
Сравнение моделей выполняется на уровне весов и графа.

### Веса
Сравнение весов:
1. веса, присутствующие только в модели A
2. веса, присутствующие только в модели B
3. количество общих весов

Совпадение значений:
4. точное совпадение (exact_equal)
5. совпадение с допуском (allclose)

Метрики различия:
6. максимальное абсолютное отличие
7. среднее абсолютное отличие
8. RMSE (среднеквадратическая ошибка)
9. косинусное сходство

Дополнительные проверки:
10. несовпадение формы
11. несовпадение типа данных

Суммирующие метрики:
12. количество полностью совпадающих весов
13. количество численно близких весов
14. количество несовпадающих весов
15. количество несовпадений формы и типов


### Графы
Сравнение графов:
Общее:
1. различия в количестве нод
2. различия в глубине графа

Различия в операциях:
3. различия в гистограмме типов операций

Сравнение нод:
4. совпадение типа операции
5. совпадение входных тензоров
6. совпадение выходных тензоров
7. совпадение топологии (in-degree, out-degree, depth)

Суммирующие метрики:
8. количество совпадающих нод
9. количество изменённых нод
10. количество изменений типа операций
11. количество изменений входов/выходов
12. количество изменений топологии
13. количество нод, присутствующих только в одной из моделей

# Пример

В файле analyzator.py приведено сравнение модели A и 4-мя другими моделями:
- модель B получена путем дообучения на 200 примерах;
- модель BB получена путем дообучения на 500 примерах;
- модель C получена путем добавления одного слоя;
- модель D получена путем изменения топологии графа;
